# ИИ-помощник для поиска поставщиков

AI-сервис для поиска, оценки и выбора поставщиков по заданной категории товаров.
Проект разработан в рамках тестового задания.

---


## Описание

Сервис помогает специалистам по закупкам:

- находить поставщиков по категории товаров и региону;
- получить ключевую информацию по поставщику;
- сравнить предложения;
- автоматически рассчитать рейтинг поставщиков;
- получить рекомендации на основе AI-анализа;
- принять решение в выборе быстрее.

## Установка и запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/ErmakovEvgen/Suppliers_service
cd Suppliers_service
```
### 2. Создание виртуального окружения

```bash
python -m venv venv
```
### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка API

Создайте файл .env в корне проекта:
```text
OPENAI_API_KEY=<api-ключ>
OPENAI_BASE_URL=<url>
```
Данные для заполнения .env переданы сотруднику HR.

### 5. Запуск приложения
Для запуска сервиса в терминале используйте:
```terminal
streamlit run main.py
```
После запуска приложение будет доступно по:
```text
http://localhost:8501
```

## Основные возможности

### Поиск поставщиков

Пользователь указывает категорию товара и регион поиска.

Система подбирает подходящих поставщиков из предоставленной базы данных.

---

### Автоматическая оценка поставщиков

Для каждого поставщика рассчитывается свой итоговый рейтинг.

Оценка строится на основе 6 факторов:

| Параметр | Вес |
|-----------|------|
| Рейтинг поставщика | 40% |
| Наличие сертификата | 15% |
| Сумма для бесплатной доставки | 10% |
| Минимальная сумма заказа | 10% |
| Уровень цен | 15% |
| Условия оплаты | 10% |

Максимальный итоговый балл — 100.

---

### AI-анализ

После поиска по категории или региону у пользователя есть возможность запустить AI-анализ для помощи с выбором подходящего поставщика по кнопке. Такой подход позволяет снизить количество запросов к модели, уменьшить затраты и ускорить работу интерфейса.

На основе найденных поставщиков модель:
- анализирует ключевые данные;
- сравнивает их между собой;
- выбирает наиболее предпочтительного кандидата;
- формирует обоснование выбора;
- указывает преимущества и возможные недостатки.

---

### Взаимодействие с результатом

После поиска у пользователя есть возможность переключаться между двумя режимами отображения поставщиков:

- Карточки
- Таблица

В режиме отображения "Карточки" поставщик, набравший наибольшее количество баллов получает выделение на фоне конкурентов плашкой "🏆Рекомендуемый".
В режиме отображения "Таблица" у пользователя есть возможность сортировки, скрыть/показать выбранные столбцы, а также сохранить результат в формате CSV.

## Структура проекта

- /data/suppliers.json: - база данных с 50-ю поставщиками в разных категориях и регионах
- /services/supplier_service.py - функции поиска и расчёта итогового балла поставщика
- /services/llm_service.py - функция подключения AI-модели и промпт для её работы по генерации рекомендации выбора
- ui/components.py - Вывод информации о поставщиках в виде карточек
- ui/table.py - Вывод информации о поставщиках в виде таблицы с возможностью сортировки
- ui/info.py - Вывод информационной таблички с описанием подсчёта итогового рейтинга поставщика
- requirements.txt - Список зависимостей
- Конфигурация API-ключей хранится через переменные окружения.

## Логика работы

```text

        ↓

        ↓

        ↓
```

## Обработка ошибок
Сервис предусматривает обработку ошибок взаимодействия с AI API.

В случае:
- отсутствия интернет-соединения;
- недоступности AI-сервиса;
- ошибки авторизации;

пользователю выводится информативное сообщение без остановки работы приложения.

## Бизнес-ценность

Проект демонстрирует применение генеративного ИИ для поддержки принятия решений в закупочной деятельности.
Система объединяет:
- структурированные данные о поставщиках;
- алгоритмическую оценку по набору бизнес-критериев;
- анализ результатов с помощью LLM-модели.

Такой подход позволяет автоматизировать часть аналитической работы специалиста по закупкам и получать обоснованные рекомендации на основе имеющихся данных.
